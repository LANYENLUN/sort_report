一、操作原理

選擇排序法（Selection Sort）是一種簡單直觀的排序演算法。

核心概念：
1. 將陣列分為「已排序區」與「未排序區」。初始時，全都是未排序區。
2. 每次從未排序區中掃描尋找最小值（圖表中以紫色標示）。
3. 找到最小值後，將它與未排序區的「第一個元素」進行交換。
4. 此時，已排序區增加一個元素，未排序區減少一個。重複此動作直到全部排序完成。

二、演算法偽代碼

function selectionSort(arr):
  n = arr.length
  for i = 0 to n - 2:
    minIndex = i
    // 在未排序區尋找最小值的索引
    for j = i + 1 to n - 1:
      if arr[j] < arr[minIndex]:
        minIndex = j
        
    // 將找到的最小值與未排序區的起始位置交換
    if minIndex != i:
      swap(arr[i], arr[minIndex])

三、時間 & 空間複雜度

情況 時間複雜度 說明
最佳情況 (Best) O(n²) 即使陣列已排序，仍需要掃描未排序區尋找最小值。
平均情況 (Average) O(n²) 需要進行兩層迴圈掃描。
最壞情況 (Worst) O(n²) 執行時間與初始資料狀態無關，永遠需要 O(n²) 比較次數。
空間複雜度 O(1) 原地排序，僅需常數的額外空間。
穩定性 不穩定 ✗ 長距離交換可能會破壞相等元素的相對順序（例如 `[5, 5, 2]` 交換後變成 `[2, 5, 5]`）。
