一、操作原理

快速排序法（Quick Sort）同樣採用「分治法 (Divide and Conquer)」，是實務上最常用、平均速度最快的演算法之一。

核心概念：
1. 挑選基準 (Pivot)：從陣列中挑選一個元素作為基準值（此範例固定選擇當前區間的最後一個元素，圖表中以紫色標示）。
2. 分割 (Partition)：重新排列陣列，將所有比基準值小的元素移到基準值左邊，比基準值大的移到右邊。分割完成後，該基準值就成功「歸位」到最終正確的位置了。
3. 遞迴排序：對基準值左右兩邊的子陣列，重複執行上述步驟，直到子陣列長度為 1 或 0，排序即完成。

二、演算法偽代碼

function quickSort(arr, low, high):
  if low < high:
    // 進行分割，並取得基準值歸位後的索引
    pivotIndex = partition(arr, low, high)
    
    // 對基準值左右兩邊遞迴排序
    quickSort(arr, low, pivotIndex - 1)
    quickSort(arr, pivotIndex + 1, high)

function partition(arr, low, high):
  pivot = arr[high]  // 選擇最後一個元素為基準
  i = low - 1
  for j = low to high - 1:
    if arr[j] < pivot:
      i++
      swap(arr[i], arr[j])
  swap(arr[i + 1], arr[high])  // 將基準值放到正確位置
  return i + 1

三、時間 & 空間複雜度

情況 時間複雜度 說明
最佳情況 (Best) O(n log n) 每次挑選的 Pivot 剛好都能將陣列平分為兩半。
平均情況 (Average) O(n log n) 多數情況下效能極佳，常數因子小，實際執行速度比多數排序都快。
最壞情況 (Worst) O(n²) 陣列已經排序好，且每次都選最後一個元素當 Pivot 時發生。
空間複雜度 O(log n) 主要來自於遞迴呼叫時所需的堆疊 (Stack) 空間。最壞情況為 O(n)。
穩定性 不穩定 ✗ 在進行跨距離交換時，可能會打亂相等元素的原始相對順序。
