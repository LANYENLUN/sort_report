一、操作原理

合併排序法（Merge Sort）是採用「分治法 (Divide and Conquer)」的經典演算法，效能非常優異且穩定。

核心概念：
1. 分割 (Divide)：將陣列不斷對半切分，直到每個子陣列只剩下 1 個元素（長度為 1 的陣列本身就是已排序狀態）。
2. 合併 (Merge)：將相鄰的兩個已排序子陣列，透過比較大小，依序放入一個暫存陣列中，最後再寫回原陣列，直到合併成一個完整的已排序陣列。

二、演算法偽代碼

function mergeSort(arr, left, right):
  if left >= right: return
  
  mid = floor((left + right) / 2)
  mergeSort(arr, left, mid)       // 遞迴排序左半部
  mergeSort(arr, mid + 1, right)  // 遞迴排序右半部
  
  merge(arr, left, mid, right)    // 執行合併操作

三、時間 & 空間複雜度

情況 時間複雜度 說明
最佳情況 (Best) O(n log n) 不論資料初始狀態，都需要切分到底再合併。
平均情況 (Average) O(n log n) 每次將陣列切半需要 log n 層，每層合併花費 O(n)。
最壞情況 (Worst) O(n log n) 即使在最壞情況下，效能依然能保持 O(n log n)。
空間複雜度 O(n) 合併時需要建立與原陣列等長 (n) 的暫存空間，空間消耗較大。
穩定性 穩定 ✓ 合併過程中，若元素相等，會優先取左半邊的元素，保持原相對順序。
