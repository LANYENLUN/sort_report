一、操作原理

插入排序法（Insertion Sort）的運作方式就像是我們在打撲克牌時「理牌」的過程。

核心概念：將陣列分成「已排序」與「未排序」兩個區域。初始時，將第一個元素視為已排序。接著從未排序區逐一取出元素（Key），由右至左與已排序區的元素進行比較。若找到比 Key 大的元素，則將該元素往右平移。直到找到不大於 Key 的元素（或到達最左端）時，將 Key 插入該位置。

二、演算法偽代碼

function insertionSort(arr):
  n = arr.length
  for i = 1 to n-1:
    key = arr[i]
    j = i - 1
    // 在已排序區間由右至左尋找插入位置
    while j >= 0 and arr[j] > key:
      arr[j+1] = arr[j]  // 將較大的元素向右平移一格
      j = j - 1
    arr[j+1] = key       // 將 key 插入到正確位置

三、時間 & 空間複雜度

情況 時間複雜度 說明
最佳情況 (Best) O(n) 陣列已是排序狀態，每次只需比較 1 次，無須平移。
平均情況 (Average) O(n²) 元素隨機分布，平均需要比較與移動一半的已排序元素。
最壞情況 (Worst) O(n²) 陣列完全逆序，每次都需要比較並移動到最前端。
空間複雜度 O(1) 原地排序，僅需常數的額外空間儲存 key。
穩定性 穩定 ✓ 相等元素不會交錯，不改變原始相對順序。
