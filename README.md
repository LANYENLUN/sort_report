## 前言
本報告旨在透過實作五種經典的排序演算法（氣泡排序、選擇排序、插入排序、合併排序、快速排序），探討各演算法的運作邏輯與效能差異。專案中不僅設計了互動式的動態視覺化網頁幫助理解過程，更建立了一個純淨的實驗環境，藉由操作不同規模的數據量，客觀測量並驗證理論上的時間複雜度。

---

## 排序法原理
1. **氣泡排序法 (Bubble Sort)：**
   反覆掃描陣列，每次比較相鄰的兩個元素，若左邊大於右邊則進行交換。每一輪結束後，當前未排序部分的最大值會如氣泡般「浮」到陣列末端。
2. **選擇排序法 (Selection Sort)：**
   將陣列分為已排序與未排序兩區。每次從未排序區中掃描並找出「最小值」，然後將其與未排序區的第一個元素交換，逐步擴大已排序區塊。
3. **插入排序法 (Insertion Sort)：**
   將元素逐一取出，由右至左與已排序區間的元素比較，將較大的元素往右平移，直到找到合適的位置插入。類似於打撲克牌時的理牌動作。
4. **合併排序法 (Merge Sort)：**
   採用分治法 (Divide and Conquer)。將陣列不斷對半切割，直到子陣列長度為 1，接著再將相鄰的子陣列兩兩比較並合併，最終組合成完整的已排序陣列。
5. **快速排序法 (Quick Sort)：**
   同樣基於分治法。從陣列中挑選一個「基準值 (Pivot)」，將所有比 Pivot 小的元素移到左邊，比 Pivot 大的移到右邊（此過程稱為 Partition 分割），再對左右兩邊的子陣列遞迴執行相同操作。

## 複雜度分析
以下為五種排序演算法的理論複雜度整理：

| 排序演算法 | 最佳時間複雜度 | 平均時間複雜度 | 最壞時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **氣泡排序** | O(n) | O(n²) | O(n²) | O(1) | 穩定 |
| **選擇排序** | O(n²) | O(n²) | O(n²) | O(1) | 不穩定 |
| **插入排序** | O(n) | O(n²) | O(n²) | O(1) | 穩定 |
| **合併排序** | O(n log n) | O(n log n) | O(n log n) | O(n) | 穩定 |
| **快速排序** | O(n log n) | O(n log n) | O(n²) | O(log n) | 不穩定 |

## 實驗設計
為確保測速實驗的客觀性與準確性，本實驗採用以下設計：
* **實驗環境：** 於瀏覽器前端環境中，使用 JavaScript 高精度計時器 `performance.now()` 進行測量，單位為毫秒 (ms)。
* **測試方式：** 拔除所有與 DOM（網頁畫面）互動及 `sleep()` 動畫延遲的程式碼，採用「純演算法 (Pure Functions)」進行效能測試。
* **變因控制：** * **操作變因：** 陣列資料規模 N，分別測試 1,000、5,000、10,000、30,000 筆資料。
  * **控制變因：** 每次測量時，五種演算法皆對「完全相同的一份隨機亂數陣列」進行排序。
* **數據採樣：** 為了減少單次亂數分佈極端值或系統背景資源佔用造成的誤差，本報告共執行了 5 次完整的測速實驗，並將數據進行平均計算。

## 結果比較

以下為 5 次獨立實驗數據加總後的平均執行時間（單位：毫秒 ms）：

| 排序演算法 \ 資料規模 N | N = 1,000 | N = 5,000 | N = 10,000 | N = 30,000 |
| :--- | :--- | :--- | :--- | :--- |
| **氣泡排序 Bubble Sort** | 0.70 | 18.66 | 68.34 | 560.66 |
| **選擇排序 Selection Sort**| 0.58 | 15.14 | 52.42 | 453.02 |
| **插入排序 Insertion Sort**| 0.14 | 5.14 | 18.68 | 156.24 |
| **合併排序 Merge Sort** | 0.26 | 2.06 | 3.14 | 9.22 |
| **快速排序 Quick Sort** | 0.12 | 0.26 | 0.60 | 1.68 |

**效能差異觀察：**
從實驗數據中可以明顯看出，屬於 O(n²) 等級的氣泡、選擇、插入排序，其執行時間會隨著 N 增加而呈現劇烈的指數型增長；當 N 來到 30,000 時，氣泡排序的平均時間已高達 560 毫秒。相較之下，屬於 O(n log n) 的合併與快速排序展現了極佳的延展性，即使在 N = 30,000 時，快速排序依然僅需約 1.68 毫秒即可完成。
此外，在 O(n²) 家族中，插入排序的實際表現明顯優於氣泡與選擇排序，這印證了插入排序在常數項極小及部分有序情況下具備優勢的理論。
